# Software Testing Project – Mozilla Firefox Browser

## Project Overview
This project focuses on performing manual testing on the Mozilla Firefox Browser to verify its functionality, usability, security, compatibility, and performance. The goal of this project is to ensure that the browser works correctly across different scenarios and provides a reliable user experience.

The project includes the creation of a detailed Test Plan, execution of multiple Test Cases, and documentation of Defect Reports identified during testing.

---

## Objectives
- To understand the software testing process.
- To design and execute test cases.
- To identify and report software defects.
- To verify browser functionality under different conditions.
- To ensure usability, security, and compatibility of the application.

---

## Testing Types Covered
The following testing techniques were applied in this project:

### 1. Functional Testing
Verifies that all features of the browser work according to the requirements.

Examples:
- Search functionality
- Download progress
- Tab management
- Application installation
- Multiple app usage

### 2. Security Testing
Ensures that the application protects user data and prevents unauthorized access.

Examples:
- Login validation
- Password security
- Private browsing data protection
- Location access verification

### 3. Localization Testing
Ensures the browser works correctly in different regions and languages.

Examples:
- Language settings
- Currency settings
- Theme customization

### 4. Compatibility Testing
Checks whether the application works on different devices and platforms.

Examples:
- Smartphone
- Tablet
- Windows PC
- Browser compatibility

### 5. Usability Testing
Evaluates how easy and user-friendly the browser interface is.

Examples:
- Menu navigation
- Icon visibility
- Bookmark functionality
- Private window access

### 6. Load Testing
Verifies the performance of the browser under expected user load.

Examples:
- Page refresh
- Page reload
- Forward and backward navigation
- Page loading time

## Defect Reporting
Several defects were identified during testing. Each defect report includes:

- Defect ID
- Module
- Severity
- Priority
- Steps to Reproduce
- Expected Result
- Actual Result
- Build Version

Example defects include:
- Cursor appearing on the homepage without text fields.
- New tab not opening when clicked.
- Bookmark duplication issue.
